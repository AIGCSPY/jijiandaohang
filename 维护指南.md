# G123.cn 导航站 · 维护指南

本文说明如何新增 / 修改分类与工具。日常只需改 **`categories.js`**，再跑一条命令同步 SEO 导航。

---

## 一、新增工具（最常用）

### 1. 编辑 `categories.js`

打开文件，找到对应分类的 `links` 数组，在末尾追加：

```javascript
{
  "name": "工具名称",
  "url": "https://example.com/",
  "desc": "一句话简介（悬停气泡 + SEO 描述）"
}
```

**示例**：给「写作」分类加一个工具

```javascript
{
  "id": "writing",
  "name": "写作",
  "links": [
    // ……已有条目……
    {
      "name": "新工具名",
      "url": "https://example.com/",
      "desc": "用来做什么"
    }
  ]
}
```

| 字段 | 要求 |
|------|------|
| `name` | 页面上显示的名称 |
| `url` | 完整地址，以 `https://` 开头 |
| `desc` | 简短中文描述，建议 15～25 字 |

改完保存。

### 2. 同步 SEO 导航（必做）

在项目根目录执行：

```bash
node scripts/build-seo-nav.js
```

作用：根据 `categories.js` 重新生成 `index.html` 里 `<nav id="seoNav">` 的静态 HTML，保证「查看源代码」和搜索引擎能看到全部链接。

> **别忘了这一步**：只改 `categories.js` 不跑脚本，页面上能看到新工具，但 HTML 源代码里的 SEO 导航不会更新。

### 3. 本地预览

```bash
cd 你的目录后
python3 -m http.server 8080
```

浏览器打开 http://localhost:8080/ ，切换到对应分类 tab，悬停确认文案正常。

### 4. 部署上线

至少上传这两个文件：

- `categories.js`
- `index.html`（跑过 build 脚本之后的版本）

---

## 二、修改已有工具

在 `categories.js` 里找到对应条目，改 `name` / `url` / `desc`，保存后同样执行：

```bash
node scripts/build-seo-nav.js
```

---

## 三、删除工具

从对应分类的 `links` 数组里删掉该对象，再执行 `node scripts/build-seo-nav.js`。

---

## 四、新增一整类分类（较少用）

当前共 **12** 类：精选、写作、绘画、视频、编程、音乐、办公、设计、语音、营销、搜索、学习。

1. 在 `categories.js` 的 `window.G123_CATEGORIES` 数组末尾增加：

```javascript
{
  "id": "新分类英文id",   // 如 podcast，用于 URL 锚点 #cat=podcast
  "name": "播客",          // tab 上显示的中文名
  "links": [ /* 工具列表 */ ]
}
```

2. 执行 `node scripts/build-seo-nav.js`

3. 同步更新 **`submit/index.html`** 里「推荐分类」下拉框，加一行：

```html
<option value="播客">播客</option>
```

4. 本地预览 tab 是否多出一项，锚点 `/#cat=新分类英文id` 能否直达

---

## 五、只改「精选」里的工具

编辑 `categories.js` 中 `"id": "featured"` 那一组的 `links` 即可，步骤同一、二节。

---

## 六、默认收藏（新用户看到的 15 个图标）

文件：**`favorites-defaults.js`**

与分类数据独立。改这里的列表只影响「从未自定义收藏」的新访客，改完无需跑 build 脚本。

---

## 七、数据流一览

```
categories.js
    ├── app.js 读取 → 分类 tab、工具列表、悬停文案、星标收藏
    └── node scripts/build-seo-nav.js → index.html 内 SEO 导航（静态 HTML）

favorites-defaults.js → 默认收藏（独立）
```

---

## 八、常用命令速查

| 操作 | 命令 |
|------|------|
| 同步 SEO 导航 | `node scripts/build-seo-nav.js` |
| 本地演示 | `python3 -m http.server 8080` |
| 检查 JS 语法 | `node --check app.js` |

---

## 九、相关文件

| 文件 | 作用 |
|------|------|
| `categories.js` | **分类与工具主数据（改这里）** |
| `scripts/build-seo-nav.js` | 生成 SEO 导航到 index.html |
| `index.html` | 首页 + 静态 SEO 导航 |
| `app.js` | 交互逻辑（一般不用动） |
| `favorites-defaults.js` | 默认收藏 15 个 |
| `submit/index.html` | 用户提交页的分类下拉 |

---

## 十、备忘

- 站点部署在域名根目录，资源路径为 `/style.css` 这种，**没有** `/g/` 前缀
- SEO 导航在页面上不可见（CSS 隐藏），但必须在 HTML 源代码里存在
- 用户个人收藏存在浏览器本地，与 `categories.js` 无关
